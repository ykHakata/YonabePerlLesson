# Lesson1 Perl をつかって HTML をはきだす仕組み

さくらのサーバーで Perl のスクリプトを走らせるにはどうすればいいのか？

    サーバーコントロールパネル -> サーバ情報とパスワード -> サーバー情報の表示

    さくらのサーバーには数種類のスクリプト言語が利用できるようになっている
    スクリプト言語を走らせる前にサーバー(運営会社)によって制約があるので
    事前に確認しておく。

    CGI・PHP・SSI について
    https://help.sakura.ad.jp/app/answers/detail/a_id/2104/c/232

    この辺りの情報は一般的なプログラミングの知識に明るくないと
    理解が難しい内容になっており、サーバーの OS も FreeBSD (UNIX系)
    UNIX系の知識が必要になってくる、開発するPC は UNIX系のOSの方が
    効率が良く、初心は MacOS が現実的な選択肢になる

HTML をはきだすだけの Perl スクリプトを作成してみる

    Lesson2/Sample/lesson2.cgi を参考に
    Lesson2/Note/lesson2.cgi を完成させる

    HTML のヘッダ以外に HTTP のヘッダが記載されていることに注意

    完成のち、手元のターミナルでファイルを実行する

    $ cd ~/yonabe20151121/Lesson2/Note
    (手元のPCによって、パスは違うので注意)

    $ perl lesson2.cgi

Perl スクリプトをアップロードしてみる

    Lesson1 の document.md を参考に lesson2.cgi をアップロードしてみる

    先ほどの手順で表示しようとすると Internal Server Error となり
    表示できない。

    ファイルの実行権限(パーミッション)を変更する必要がある

    -----
    [注意] 手元でパーミッションを 755 に変更
    $ chmod 755 lesson2.cgi
    としても、アップロードすると権限が変わっているので
    結局、ファイルマネージャー側で変更することになる
    -----

    ファイルマネージャー画面、lesson2.cgi ファイルにチェックし
    左上の [操作] -> [プロパティ] -> [実行] に全部チェック 755 にする -> OK

    もう一度表示させてみる。

まとめ

    一般的にいうホームページは HTML 形式で記述された
    シンプルテキストファイルを WEB サーバーを使ってはきだしているだけに対して、
    いわゆる WEB アプリはWEB サーバーを通じて、
    コードが書かれたファイルを読み込みプログラミングの実行結果を
    いわゆる Print などの標準出力を通じて出力されたものを WEB を使って
    はきだしている。
